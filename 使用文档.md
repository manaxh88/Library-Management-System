# 图书管理系统使用文档

## 1. 项目说明
本项目为前后端分离的图书管理系统：
- 后端：Spring Boot 3（Java 17）
- 前端：Vue 3 + Vite + TypeScript
- 数据库：MySQL
- 推荐模块：Python（混合推荐算法，按需启用）

角色说明：
- 管理员（role=0）：用户/图书管理、统计分析、推荐配置、续借审批
- 教师（role=1）：批量借阅、推荐查看、续借申请管理
- 学生（role=2）：个性化推荐、借阅提醒、借阅历史

## 2. 环境要求
请先安装以下软件：
- JDK 17
- Maven 3.9+
- Node.js 18+
- MySQL 8.x
- Python 3.10+（仅在使用推荐算法时需要）

## 3. 启动前配置
### 3.1 后端配置文件
文件位置：`backend/src/main/resources/application.yml`

关键配置项：
- 服务端口：`server.port: 8080`
- 数据库连接：
  - `spring.datasource.url: jdbc:mysql://localhost:3306/Library_Management_System...`
  - `spring.datasource.username: root`
  - `spring.datasource.password: 123456`

请根据你的本地环境修改数据库用户名和密码。

### 3.2 JWT 密钥
配置项：`app.jwt.secret`

当前示例值为：
`replace-with-a-long-secret-key-at-least-32-bytes`

建议替换为你自己的随机长字符串（至少 32 字节），例如：
`app.jwt.secret: "your-very-long-random-secret-key-xxxxxxxx"`

### 3.3 数据库准备
当前配置为：`spring.jpa.hibernate.ddl-auto: none`

这表示后端不会自动建表，你需要提前在 MySQL 中准备好业务表结构和初始数据（包括用户、图书、借阅等表）。

## 4. 启动步骤
### 4.1 启动后端
在项目根目录打开终端后执行：

```bash
cd backend
mvn spring-boot:run
```

启动成功后，后端地址：
- `http://localhost:8080`
- 健康检查：`GET http://localhost:8080/health`，返回 `OK`

### 4.2 启动前端
新开一个终端执行：

```bash
cd frontend
npm install
npm run dev -- --host 0.0.0.0 --port 5173
```

启动成功后，访问：
- `http://localhost:5173`

说明：
- 前端请求基地址是 `/api`
- Vite 代理会把 `/api/*` 转发到 `http://localhost:8080/*`

## 5. 登录与角色访问
### 5.1 登录接口
后端登录接口：
- `POST /login`

在前端代理下，实际调用路径为：
- `POST /api/login`

登录成功后会返回 JWT，前端会自动把 token 放入请求头：
- `Authorization: Bearer <token>`

### 5.2 账号说明
项目不提供内置测试账号。

请先在你本地数据库中准备好可用的用户数据（管理员/教师/学生），再使用对应账号登录。

### 5.3 角色页面入口
- 管理员：`/admin`、`/admin/users`、`/admin/books`、`/admin/statistics`
- 教师：`/teacher/batch-borrow`、`/teacher/recommendations`、`/teacher/renew-requests`
- 学生：`/student/recommendations`、`/student/reminders`、`/student/borrow-history`

## 6. 推荐算法模块（可选）
仅在使用个性化推荐重算功能时需要。

### 6.1 安装 Python 依赖
```bash
pip install -r backend/recommendation/requirements.txt
```

依赖主要包括：
- `scikit-learn`
- `numpy`

### 6.2 推荐相关配置
文件：`backend/src/main/resources/application.yml`

配置项：
- `app.recommendation.python-command: python`
- `app.recommendation.python-script: recommendation/hybrid_recommender.py`
- `app.recommendation.default-top-n: 20`

如果你的 Python 命令不是 `python`（如 `py` 或虚拟环境路径），请调整 `python-command`。

## 7. 常见问题排查
### 7.1 前端启动后无法请求后端
- 检查后端是否已启动在 `8080`
- 检查 `frontend/vite.config.ts` 的代理目标是否为 `http://localhost:8080`
- 检查浏览器控制台是否有 401（未登录/Token 失效）

### 7.2 后端启动报数据库连接失败
- 确认 MySQL 服务已启动
- 确认数据库名 `Library_Management_System` 已创建
- 检查 `application.yml` 中用户名/密码是否正确

### 7.3 登录失败
- 确认请求路径为 `/api/login`（前端）或 `/login`（后端直连）
- 确认数据库存在对应用户
- 检查输入的用户名、密码是否与数据库中的用户数据一致

### 7.4 推荐功能报错（Python 相关）
- 确认已安装 `backend/recommendation/requirements.txt` 依赖
- 在后端运行目录下确认 `recommendation/hybrid_recommender.py` 路径可访问
- 检查 `app.recommendation.python-command` 是否可执行

## 8. 开发建议
- 前后端开发时建议分别使用两个终端常驻运行（`mvn spring-boot:run` 和 `npm run dev`）
- 首次联调优先验证：登录 -> 首页 -> 图书列表 -> 对应角色功能页
- 生产环境请务必修改数据库凭据与 JWT 密钥，并关闭不必要的调试日志
